# 物料设计

## 设计目标

物料系统用于统一管理游戏中出现的实体、地形、资源和结构资产。它既服务当前可玩版本，也为后续 AI 生成、人工审核、资产替换和版本回滚留出空间。

## 物料分类

当前物料分类由 `MaterialCategory` 定义:

- `enemy`: 敌方单位，包括刀兵、枪兵、骑兵、攻城车等。
- `tower`: 防御单位，包括箭塔、炮台、冰塔、投石车、火箭塔等。
- `terrain`: 地图地形，包括山地、树木、草地、河流。
- `resource`: 资源图标，包括木材、石料、粮草、金币。
- `structure`: 固定结构，包括城池、城墙。
- `ui`: UI 装饰、面板和图标。

## 物料等级

物料等级由 `MaterialLevel` 定义:

- `basic`: 基础物料，优先使用程序化或稳定手工资产。
- `intermediate`: 中级物料，视觉复杂度更高，可使用 AI 辅助生成。
- `advanced`: 高级物料，通常需要 3D 模型、动作或更高质量贴图。

## 物料来源

物料来源由 `MaterialSource` 定义:

- `procedural`: 程序化几何体或代码绘制资产，最稳定，适合作为默认回退。
- `ai-2d`: AI 生成 2D 图像，适合图标、角色立绘、资源图和 UI 背景。
- `ai-3d`: AI 生成 3D 模型，适合攻城车、投石车、城池和高级塔。

任何 `ai-2d` 或 `ai-3d` 资产都必须保留生成提示词和生成来源，方便后续复现、替换和审核。

## 数据字段

核心字段:

- `id`: 稳定唯一标识，例如 `tower-arrow-lv1`。
- `name`: 中文显示名。
- `category`: 物料分类。
- `level`: 物料等级。
- `source`: 物料来源。
- `prompt`: 生成提示词或设计描述。
- `description`: 游戏内用途说明。
- `entityType`: 与游戏实体类型或渲染工厂的映射。
- `imageUrl`: 2D 图片地址或 data URL。
- `modelUrl`: 3D 模型地址，优先 GLB。
- `skeletonActions`: 可用动作列表，例如 `idle`、`attack`、`load`。
- `createdAt`、`updatedAt`: 时间戳。

## 命名规范

物料 `id` 使用 kebab-case，按分类和用途组织:

- 敌人: `enemy-sword`、`enemy-cavalry`、`enemy-ram`
- 防御塔: `tower-arrow-lv1`、`tower-cannon-lv2`
- 地形: `terrain-mountain`、`terrain-river`
- 资源: `resource-wood`、`resource-gold`
- 结构: `structure-castle`、`structure-wall`

等级型资产使用 `lv1`、`lv2`、`lv3` 后缀，避免中文等级进入 ID。

## 视觉风格

整体风格是中国古代战争题材，优先满足清晰识别和游戏可读性。

通用要求:

- 等距视角下轮廓清楚。
- 阵营、功能和等级差异明显。
- 防御塔升级后体量、层数、装饰或武器规模应逐级增强。
- 敌人单位在移动中不能和地形混淆。
- 资源图标使用高对比剪影，适合小尺寸显示。

2D 资产要求:

- 背景透明或容易抠图。
- 主要对象居中，占画面 70% 左右。
- 避免复杂文字和细碎纹样。
- UI 图标在 48px 到 96px 尺寸下仍可辨认。

3D 资产要求:

- 优先 GLB 格式。
- 模型朝向默认面向路径前进方向。
- 多边形数量控制在浏览器实时渲染可接受范围。
- 攻击类资产预留 `idle`、`attack` 动作语义，即使当前版本用程序化动画替代。

## 默认物料清单

敌方单位:

- 刀兵: 基础近战单位。
- 枪兵: 长柄武器单位。
- 骑兵: 高速突击单位。
- 炮兵: 远程拆塔单位。
- 大力士: 重甲肉盾单位。
- 巫师: 范围法术单位。
- 攻城车: 破门攻城器械。
- 投石车: 远程重型攻城器械。

防御塔:

- 箭塔 Lv.1-Lv.3: 单体快速攻击。
- 炮台 Lv.1-Lv.3: 范围爆炸伤害。
- 冰塔 Lv.1-Lv.3: 减速控制。
- 投石车 Lv.1-Lv.2: 重型守城器械。
- 火箭塔 Lv.1: 多目标或爆发型扩展塔。

地形与结构:

- 山地、树木: 阻挡和视觉层次。
- 草地: 主要可建造地块。
- 河流: 路径和阻挡边界。
- 城池: 防守核心。
- 城墙: 防御结构扩展。

资源:

- 木材: 建造资源。
- 石料: 防御升级资源。
- 粮草: 兵力维持资源。
- 金币: 通用资源。

## 生成流程

2D 生成:

```text
物料界面选择物料 -> 输入/复用 prompt -> 服务端代理 DRAW_API_KEY -> 返回图片 -> 写入 imageUrl
```

3D 生成:

```text
物料界面选择物料 -> 提交 prompt -> 记录 jobs 表 -> 定时查询 -> 完成后写入 modelUrl/imageUrl
```

## 审核和回滚规划

当前版本直接写入物料记录。后续建议增加:

- `status`: `draft`、`approved`、`rejected`。
- `version`: 物料版本号。
- `provider`: 生成供应商。
- `generationParams`: 模型、尺寸、种子、风格参数。
- `reviewNotes`: 审核备注。

这样可以让 AI 生成资产先进入草稿区，通过人工确认后再用于正式关卡。
