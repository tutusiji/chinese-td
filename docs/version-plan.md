# 开发版本计划

## 版本目标

项目采用小步迭代方式推进，每个版本必须保持可构建、可部署、可回滚。线上主分支为 `main`，部署目录为 `/opt/chinese-td`。

## v0.1 基础可玩版

状态: 已完成

目标:

- 建立 Vite + TypeScript 前端工程。
- 实现基础等距塔防地图、敌人移动、建塔、升级、出售、波次和胜负判定。
- 实现箭塔、炮台、冰塔三类核心防御塔。
- 建立基础 HUD、关卡选择和地图配置。

验收标准:

- `npm run build` 成功。
- 本地可以完成一局防守模式。
- 地图、塔、敌人数据从配置文件读取。

## v0.2 中国古代题材扩展

状态: 已完成

目标:

- 增加雁门、剑门、山海等主题地图。
- 增加刀兵、枪兵、骑兵等敌人类型。
- 建立中国古代城池、城墙、箭塔、炮台和地形视觉风格。
- 增加攻城模式雏形和 AI 防守布局。

验收标准:

- 关卡选择可以切换地图和模式。
- 防守模式和攻城模式都能进入游戏循环。
- 主要实体都有可见表现和基础动画。

## v0.3 物料管理与生成链路

状态: 已完成基础能力，继续完善

目标:

- 建立 SQLite 物料表和生成任务表。
- 提供物料管理界面，支持查看和生成 2D/3D 资源。
- 接入 2D 图像生成代理。
- 接入混元 3D 和 TokenHub 3D 任务提交/查询。
- 生成成功后自动回写物料 `image_url`、`model_url`。

验收标准:

- 真实 API Key 只存在 `.env` 或服务器环境变量。
- 缺失 key 时接口返回明确配置错误。
- 已完成任务能在物料界面看到结果。

## v0.4 部署与文档完善

状态: 当前版本

目标:

- 推送到 `github.com/tutusiji/chinese-td`。
- 在 `huoshan` 服务器部署到 `joox.cc:7501`。
- 使用 pm2 管理生产进程。
- 补齐 README、架构设计、版本计划、物料设计和工具文档。
- 清理仓库中的硬编码 key 和敏感信息。

验收标准:

- GitHub push protection 不再拦截提交。
- `pm2 status chinese-td` 为 online。
- `curl http://127.0.0.1:7501/api/health` 返回 `{"ok":true}`。
- 文档能指导新开发者完成本地启动和服务器部署。

## v0.5 玩法平衡与体验打磨

状态: 计划中

目标:

- 调整各地图路径长度、初始金币、敌人血量和防御塔价格。
- 增加暂停、倍速、重新开始、关卡结算和失败复盘。
- 提升移动端布局适配。
- 增加资源产出和攻城单位编队策略。

验收标准:

- 每张地图至少有一个稳定通关策略。
- 主要 UI 在桌面和移动端无明显遮挡。
- 攻城模式有明确目标和胜负反馈。

## v0.6 资产管线产品化

状态: 计划中

目标:

- 增加物料审核状态: draft、approved、rejected。
- 支持上传自有图片和 GLB 模型。
- 增加生成参数保存，包括模型、尺寸、种子、提示词版本。
- 将大资源移动到对象存储或 CDN。

验收标准:

- 物料有完整来源记录。
- 线上包体不因生成素材无限膨胀。
- 可回滚到任意已审核物料版本。

## v0.7 账户与排行榜

状态: 计划中

目标:

- 启用用户表和登录能力。
- 增加通关记录、地图排行榜和模式排行榜。
- 保存玩家进度和自定义物料选择。

验收标准:

- 用户数据和排行榜数据有迁移脚本。
- 关键接口有鉴权。
- 排行榜可以按地图、模式、时间范围筛选。
